Career Readiness — Percentage Algorithm Design

How to fairly distribute freed weight from removed career courses to outside courses

The Problem: Equal Split is Unfair
❌ Equal Split (naive)
Remove Stat & Prob (10%) → add 5 outside courses
2% React Course 2% System Design 2% Git & DevOps 2% Docker 2% AWS
10% ÷ 5 = 2% each — a 40-lesson course worth 2% is meaningless
✅ Lesson-Count Proportional
Same removal → same 5 courses, weighted by their lesson count
4.0% System Design (40 lessons) 3.0% AWS (30 lessons) 1.5% Git & DevOps (15 lessons) 0.8% Docker (8 lessons) 0.7% React (7 lessons)
10% split by lesson count — heavier courses carry more weight
The Algorithm — 4 Steps
1

Career courses → original weights

Look up each kept career course in career_skills.weight. Sum them → careerTotalWeight.

Example: kept Python(30) + SQL(30) + PowerBI(10) → careerTotalWeight = 70

2

Freed weight = what was removed

Removed courses took their weight out of the 100-point budget.

freedWeight = 100 − careerTotalWeight

Example: 100 − 70 = freedWeight = 30

3

Outside courses → lesson-count proportional

Each outside course's weight =
freedWeight × (its lessons / total outside lessons)

A 40-lesson course among 100 total outside lessons gets 40% of the freed 30 = 12 points.

4

Final readiness

Always divides by 100 — the original career budget.

readiness = (careerSum + outsideSum) / 100

Complete everything → always 100%. Add an incomplete outside course → score drops honestly.

Removed: ML & DL (20%) and Stat & Prob (10%) → freedWeight = 30
Added 3 outside courses. All courses 100% complete in this example.

Key insight: If you only finish Python + SQL + PowerBI (no outside courses done yet) → readiness = 70%. Each outside course you complete pushes it toward 100%. Adding more outside courses without removing more career courses → they get 0 weight (freedWeight stays 30, just redistribution changes).

Edge Cases — All Handled
No outside courses
Remove courses, add nothing

Max readiness = careerTotalWeight / 100. Removed ML&DL+Stat&Prob → max = 70%. Honest ceiling — you've left a skill gap.

Add outside, remove nothing
freedWeight = 0

Outside courses get 0 weight. Career is already 100 points — no room. They show in the list but don't inflate the score.

No lesson count available
courseProgressMap missing

Fallback: treat all outside courses as equal weight. freedWeight / outsideCourses.length — equal split only as last resort.
